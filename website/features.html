Current Features

Features that are currently implemented

Ray-tracing engine

The game engine uses a ray-tracing engine to render 3D scenes

0-poly Spheres and 1-poly Rectangles

Spheres are rendered as 0-polygon shapes and rectangles are rendered as 1-polygon shapes for efficiency

Textures

Spheres and rectangles can be textured with solid colors or PNGs (PNG texturing is only tested on rectangles)

Custom PNG Reading Library

Z3D features a custom library to read PNGs from binary to usable RGB pixel array

Character Controller

Functionality to control camera movement through your keyboard and mouse

Basic Physics Engine

A basic physics engine which allows you to apply forces and gravity.

ASM-WS

A webserver written in x86 assembly that can handle GET and POST requests

Zest

Zest, as of now, is merely a configuration file that transpiles to Z3D compatible Zig. I hope to extend Zest's minimal functionality to be a full programming language in the future.

Upcoming Features

In order of priority

Documentation

This project is currently severely underdocumented. I plan on writing a good amount of documentation for all sub-projects

Custom Event Handlers

A custom event handler will allow you to add your own functions to run on specific player events (mouse clicks, keypress, etc.)

Better Meshes

Triangle mesh objects currently require a lot of work to be usable.

Image type support

Custom engines for more image filetypes like JPG and BMP to allow for more flexible texturing

Audio

Engines for audio filetypes like MP3 and WAV

Improved Zest

Zest currently functions as almost a configuration language. I'd like to upgrade it to function as a full-fledged programming language.

3D Models

Support for rendering custom OBJ files (or other 3D object file formats)

ASM-WS Improvements

Functionality like reading POST request data, handling PUT, DELETE, etc. methods, and executing files on GET methods
